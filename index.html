The capstone project has allowed me to showcase my interests and skill sets in the most meaningful way. It provided the freedom to enhance specific artifacts creatively and methodically, reflecting my unique approach to software development. My portfolio includes an Android application for tracking users' weight goals and an OpenGL application for rendering a 3D scene.

For the first artifact, I focused on UI/UX design, creating modern, responsive layouts compatible with various devices which adheres to the common requirements of modern day app development. For the second artifact, I expanded on the Graphics Programming coursework by implementing raycast rendering, an unconventional technique compared to modern rasterization. Raycast rendering involves shooting a ray from each screen pixel into a scene and using the color of the first hit object to determine the pixel color. Contrary to this, modern graphics uses rasterization which utilizes vertices and triangles to render a model. My implementation of raycasting involves an optimized version of the DDA (Digital Differential Analyzer) algorithm to efficiently traverse a voxel grid, or set of 3D cubes and determine intersections. There is one ray per screen pixel and each ray intersection takes the voxel color and sets the pixel as that color. Texture rendering is also implemented but for simplicity sake, it is a nested version of raycasting on a 2D grid instead of a 3D grid. This fits well into the course outcome of algorithms and datastructures as well as shows my ability to understand and program complex processes to fulfill requirements.

The third and final artifact is an AWS-hosted backend for the Android application. One limitation of the original application was its inability to persist user data across devices, restricting users to tracking their weight on a single device. To address this, I migrated the backend to AWS, leveraging its cloud platform to provide a scalable and accessible solution. Using AWS API Gateway, I developed a REST API capable of handling HTTP requests like GET, POST, and PUT. These requests trigger AWS Lambda functions, which interact with a DynamoDB database to retrieve or store data. This solution not only enables data persistence across devices but also demonstrates my ability to implement modern, scalable, and cost-effective cloud-based systems. By utilizing serverless technologies, the application can dynamically adjust to user demand without requiring dedicated server management, reducing costs and improving scalability.
