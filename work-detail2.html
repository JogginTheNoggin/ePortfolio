Artifact 3: Amazon Web Service Backend

From SQLite to Cloud-Based Database

Narrative

The artifact for this milestone is an Amazon Web Services (AWS) API integrated into an Android application. The Android application was initially designed during a summer course on mobile applications in May. We used an SQLite database to store our data. However, a significant limitation of this approach is that users can only access data on their own devices. Consequently, they cannot log in to their accounts from another device. To address this issue, my goal was to create a database equivalent to the SQLite version but accessible over the internet. I achieved this by leveraging the power of cloud computing, specifically using a serverless architecture.


A serverless architecture allowed me to focus on the application logic while delegating server management and scaling to Amazon, the service provider. The AWS services I used include API Gateway, Lambda, and DynamoDB. API Gateway enables me to create a RESTful API to access other AWS services. Lambda provides the logic to handle database calls, which then interact with the DynamoDB database. Fortunately, AWS uses HTTPS, ensuring that data sent over the network is encrypted. Additionally, I chose to use SHA-256 hashing to store only encrypted passwords at rest. This demonstrates an awareness of security principles and practices. Furthermore, sensitive information is kept in the request body to avoid being logged. I have also configured throttling for both API Gateway and the database to prevent the service from being spammed or becoming unmanageable. These four concepts—HTTPS, encryption at rest, request body storage, and throttling—offer basic security functionality.

Unfortunately, although the database and API were completed, the Android integration is coded but not fully functional. Android requires requests to be asynchronous, and I have attempted to address this issue, but it still fails to find the URL resource. That said, the API has been fully tested using Postman, and I have attached test commands for your verification. Android integration and encryption posed the biggest challenges. I initially attempted to use bcrypt by compiling the library in an EC2 instance and adding it to my Lambda functions, but it seemed unsupported. Therefore, I opted to use hashlib instead.
